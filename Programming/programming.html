Programming Languages

On the previous page we discussed the various hardware devices and how they are connected together. However, for a computer to perform a computation it must first be programmed, which is achieved through a programming language. There are different types of languages each with their own niche. However, the most distinctive feature about a language is how deeply it interacts with the hardware of the computer.

A low level language (LLL) communicates with the hardware directly. An example of this is assembly which can access specific registers. A use case for a LLL may be to program a device driver which is used to allow a device to communicate with the OS. By using a LLL such as assembly, faster performance can be achieved as no compilation is necessary.

On the other hand, a high level language (HLL) abstracts away the complexities of the hardware from the user by using English like keywords to write programs. This makes writing and maintaining software easier. Due to this many real-world applications are written using HLLs. Finally, hardware abstraction isn’t the only feature that separates a language but also the types of paradigms it offers. This is a big topic and is discussed in depth on this page.

Here is an example of a program that counts all the numbers from 1 to n written in two different languages. Notice that the program written in python requires less code compared to the java implementation. This is due to the nature of the language and the type of paradigms they offer.

n = input("What number do you want to sum to: ")
s = 0
for i in range(1,n+1):
s+=i
print("The sum of all the numbers from
1 to " + str(n) + " is " + str(s))
